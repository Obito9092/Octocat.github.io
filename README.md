# GitHub Pages for Portfolio Website

## Introduction
This guide provides step-by-step instructions on how to enable GitHub Pages and publish your portfolio website using the GitHub repository.

## Steps to Enable GitHub Pages

### 1. Set Up Your Repository
- Make sure your repository is named `username.github.io` (replace `username` with your GitHub username).

### 2. Create Your Website Files
- Add your HTML, CSS, and JavaScript files to the repository. Make sure your main file is named `index.html` as GitHub Pages serves this file by default.

### 3. Enable GitHub Pages
- Navigate to your repository on GitHub.
- Click on the `Settings` tab.
- Scroll down to the `Pages` section in the sidebar.
- Under the `Source` section, select the branch you want to use (usually `main`) and the `/ (root)` folder.
- Click `Save`.

### 4. Access Your Published Site
- Your site will be published at `https://username.github.io` after a few minutes.
- You can access it via this URL.

### 5. Custom Domain (Optional)
- If you want to use a custom domain, add your domain to the `Custom domain` field and follow the instructions provided to set up DNS records.

## Conclusion
Congratulations! You have successfully published your portfolio website using GitHub Pages. For more customization options, check the [GitHub Pages documentation](https://pages.github.com/).
